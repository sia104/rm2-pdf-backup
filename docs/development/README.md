# Development Docs

These documents are mainly for contributors and release/process review rather than day-to-day operators.

- `test-plan.md`: repository testing strategy and definition of done.
- `roadmap.md`: phased delivery plan and future enhancements.
- `agent-workflow.md`: agent-led repository workflow.
- `dev-readiness-report.md`: static development-readiness workflow notes.
- `rpi-install.md`: older Raspberry Pi installation notes kept for contributor reference.
- `rpi-dev-systemd.md`: validation service/timer notes for the development profile.
- `raw-copy-diagnostic.md`: raw-copy validation workflow notes.
- `run-local-rpi-test.md`: run-local validation workflow notes.
- `run-local-rpi-notes.md`: run-local validation notes.
- `run-local-rpi-test-extra.md`: extra run-local validation notes.
- `two-run-diagnostic.md`: two-run change-detection workflow notes.
- `rpi-selected-rerender.md`: selected-notebook rerender workflow notes.
- `pdf-publication.md`: publication behaviour notes.
- `template-notes.md`: template rendering status notes.
- `template-render-probe.md`: template probe notes.
